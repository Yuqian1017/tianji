# APP-SPEC-B.md — 产品B：天机卷·修仙学堂

> 独立于产品A的教学+游戏产品。
> 包含：境界升级·NPC教学·测验闯关·成就系统·道侣系统。
> 不包含：排盘引擎（那是产品A的事）。

---

## 定位

一个**修仙养成+知识学习**的互动应用。用户扮演一名修士，跟随四位师父修炼术数知识，通过答题升级、渡劫突破，最终飞升。

**核心循环**：学习 → 答题 → 获得灵力 → 升级 → 渡劫 → 解锁新内容 → 推进NPC关系

---

## 技术栈

- **构建**：Vite + npm
- **框架**：React
- **样式**：Tailwind CSS
- **AI**：Anthropic Claude API（NPC对话+出题+评判）
- **API Key**：用户自行输入，存 localStorage
- **持久化**：localStorage（用户档案·进度·成就）

## 测验题型

### 选择题生成（AI）

```javascript
const QUIZ_PROMPT = (domain, difficulty, realm) => `
你是玄学测验出题官。出一道${domain}领域${difficulty}难度的选择题。

当前修士境界：${realm}
只出该境界应该掌握的知识范围内的题。

返回严格JSON格式（不要markdown）：
{"q":"题目","a":["选项A","选项B","选项C","选项D"],"ans":"A","exp":"解释"}

${difficulty === '🟢必修' ? '基础概念题，考记忆和理解。' : ''}
${difficulty === '🟡进阶' ? '应用题，需要分析和判断。' : ''}
${difficulty === '🔴专研' ? '综合题，需要深度知识。' : ''}
`;
```

### 渡劫考试

```
渡劫 = 5-8道该境界必修题
要求：答对80%以上通过
失败：灵力不扣，可重新渡劫
特效：答题过程中有"雷劫"视觉效果（背景闪烁）
```

### 断卦实战题（金丹以上）

```
AI给出一个卦象 + 问事内容
玩家选择：
  1. 用神是什么？（选择题）
  2. 用神旺还是衰？（选择题）
  3. 结果吉还是凶？（选择题）
  4. 为什么？（可选：文字输入·AI评判）
```

## NPC系统

### 教学对话

```javascript
const NPC_TEACHING_PROMPT = (npc, chapter, style) => `
你现在扮演${npc.name}（${npc.title}）。
性格：${npc.personality}

当前教学内容：${chapter}
用${style === 'casual' ? '通俗有趣' : '专业严谨'}的方式讲课。

规则：
1. 保持角色性格一致
2. 用对话体讲解（师徒问答）
3. 每次讲一个小知识点，不要一次性灌太多
4. 讲完一个知识点后问修士一个小问题（检验理解）
5. 偶尔加入角色特有的口头禅和反应
`;
```

### 好感对话

```
好感度变化时触发特殊对话（来自GAME-DESIGN.md的剧情节点）。
实现方式：检查好感阈值 → 检查flag → 触发对话 → 设置flag防重复。
```

## 道侣系统

完整设计见 `GAME-DESIGN.md` 四B章。

关键实现点：
```
1. 好感度隐式积累（学习该NPC领域自动涨，不需要玩家主动操作）
2. 阶段检查（每次好感变化后检查是否触发下一阶段剧情）
3. 师父隐藏线特殊处理：
   - 好感50时埋「未济卦」线索（在师父的闲聊中提到）
   - 监听64卦查询事件（如果搜索"未济"且flag已设→触发）
   - 隐藏线从locked→解锁后进入正常阶段流程
4. 道侣试炼 = 特殊的双人渡劫考试
5. 结为道侣后永久修改NPC对话风格
```

## 数据持久化

```javascript
// 使用 localStorage
const STORAGE_KEYS = {
  profile: 'tianji-profile',
  progress: 'tianji-progress',
  achievements: 'tianji-achieve',
  relationship: 'tianji-rel',
  daily: 'tianji-daily',
  apiKey: 'tianji-api-key',
};

function loadProfile() {
  try {
    const saved = localStorage.getItem(STORAGE_KEYS.profile);
    return saved ? JSON.parse(saved) : DEFAULT_PROFILE;
  } catch {
    return DEFAULT_PROFILE;
  }
}

function saveProfile(profile) {
  localStorage.setItem(STORAGE_KEYS.profile, JSON.stringify(profile));
}
```

## 开发提示

```
1. 先做：用户档案+境界+灵力+练气期教学+基础答题。这就是可玩的MVP了。
2. NPC对话用AI生成——不需要预写所有对话，只需要Prompt+角色设定。
3. 教学内容直接从compendium的md文件中提取文本，按难度标签筛选。
4. 渡劫考试的题目用AI实时生成，不需要题库。
5. 道侣系统可以最后做——它不影响核心学习循环。
6. 持久化用window.storage，数据结构见GAME-DESIGN.md八章。
7. 师父隐藏线的"搜索未济卦触发"——在产品B中可简化为
   "学完全部课程+师父好感60+答对一道关于未济卦的特殊题"触发。
```
