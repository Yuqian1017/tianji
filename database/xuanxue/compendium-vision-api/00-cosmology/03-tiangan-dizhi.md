### 天干五合

两个天干相合，合化为另一种五行：

| 合 | 化 | 记忆 |
|----|----|------|
| 甲 + 己 → | 化土 | 甲己合土 |
| 乙 + 庚 → | 化金 | 乙庚合金 |
| 丙 + 辛 → | 化水 | 丙辛合水 |
| 丁 + 壬 → | 化木 | 丁壬合木 |
| 戊 + 癸 → | 化火 | 戊癸合火 |

**规则**：相合的两个天干，序号相差5。（甲1+己6，乙2+庚7……）

> ⚠️ **合而不一定化**：天干相合只是有合的趋势，是否真正「化」取决于月令和整体格局。这在八字论命中是关键判断点。

### 天干相冲

| 冲 | 含义 |
|----|------|
| 甲 ↔ 庚 | 阳木 vs 阳金 |
| 乙 ↔ 辛 | 阴木 vs 阴金 |
| 丙 ↔ 壬 | 阳火 vs 阳水 |
| 丁 ↔ 癸 | 阴火 vs 阴水 |
| 戊 ↔ 甲 | 不列入（同为阳但非对冲） |

> 严格来说，天干冲只有四组（甲庚、乙辛、丙壬、丁癸），即金木相冲、水火相冲。戊己土居中无冲。

### 地支藏干详解

> *"地支不像天干那么单纯。每个地支里面都藏着天干，像是一个房间里住了几个人。"——师父*

地支藏干分三层：**本气、中气、余气**

| 地支 | 本气(最强) | 中气 | 余气(最弱) |
|------|-----------|------|-----------|
| 子 | 癸 | - | - |
| 丑 | 己 | 癸 | 辛 |
| 寅 | 甲 | 丙 | 戊 |
| 卯 | 乙 | - | - |
| 辰 | 戊 | 乙 | 癸 |
| 巳 | 丙 | 庚 | 戊 |
| 午 | 丁 | 己 | - |
| 未 | 己 | 丁 | 乙 |
| 申 | 庚 | 壬 | 戊 |
| 酉 | 辛 | - | - |
| 戌 | 戊 | 辛 | 丁 |
| 亥 | 壬 | 甲 | - |

**规律**：子卯酉只有本气（纯粹），四库（辰戌丑未）藏三个干（复杂），其余藏两到三个。

> 🔮 **开发要点**：藏干在八字取十神、六爻取伏神时都需要。必须用查表法，不可推算。

三合局是三个地支合化为一种五行，力量极强：

| 三合局 | 组成 | 含义 |
|--------|------|------|
| **申子辰** → 水局 | 长生(申) + 帝旺(子) + 墓库(辰) | 水的完整生命周期 |

## 五、干支计算算法

### 年柱推算

```
🔮 实战：年柱计算

// 以公元年计算（注意：立春换年，非公历1月1日）
function yearPillar(year) {
  const stems = ['甲','乙','丙','丁','戊','己','庚','辛','壬','癸'];
  const branches = ['子','丑','寅','卯','辰','巳','午','未','申','酉','戌','亥'];
  
  // 天干：(年份 - 4) % 10  （公元4年为甲子年）
  const stemIdx = (year - 4) % 10;
  // 地支：(年份 - 4) % 12
  const branchIdx = (year - 4) % 12;
  
  return stems[stemIdx] + branches[branchIdx];
}

// 示例：2024年 → (2024-4)%10=0 → 甲, (2024-4)%12=4 → 辰 → 甲辰年 ✓

// ⚠️ 注意：此算法以立春为年分界。若日期在当年立春之前，
// 需用上一年的干支。立春时间需查表或用天文算法。
```

### 月柱推算

月柱的地支是固定的（正月寅、二月卯……），天干由年干推算：

| 年干 | 正月(寅)天干 | 口诀 |
|------|-------------|------|
| 甲/己年 | 丙寅 | 甲己之年丙作首 |
| 乙/庚年 | 戊寅 | 乙庚之岁戊为头 |
| 丙/辛年 | 庚寅 | 丙辛之年寻庚上 |
| 丁/壬年 | 壬寅 | 丁壬壬寅顺水流 |
| 戊/癸年 | 甲寅 | 若问戊癸何处起，甲寅之上好追求 |

```
🔮 实战：月柱计算

function monthPillar(yearStem, lunarMonth) {
  // lunarMonth: 1-12 (正月=1)
  const stems = ['甲','乙','丙','丁','戊','己','庚','辛','壬','癸'];
  const branches = ['寅','卯','辰','巳','午','未','申','酉','戌','亥','子','丑'];
  
  // 年干决定正月天干的起点
  const yearStemIdx = stems.indexOf(yearStem);
  const startStem = (yearStemIdx % 5) * 2 + 2; // 正月天干索引
  
  const monthStemIdx = (startStem + lunarMonth - 1) % 10;
  const monthBranchIdx = lunarMonth - 1; // 正月=寅(0)
  
  return stems[monthStemIdx] + branches[monthBranchIdx];
}

// ⚠️ 关键：月柱以节气分界，不是农历初一！
// 正月从立春开始，二月从惊蛰开始……
// 节气时间必须查表，精确到分钟级
```

### 日柱推算

> ⚠️ **日柱无法用简单公式推算**，必须查万年历。虽有近似算法，但因历法调整存在误差。生产环境务必使用查表法或经过验证的天文算法库。

### 时柱推算

时辰地支是固定的（子时23-1点……），天干由日干推算：

| 日干 | 子时天干 | 口诀 |
|------|---------|------|
| 甲/己日 | 甲子 | 甲己还加甲 |
| 乙/庚日 | 丙子 | 乙庚丙作初 |
| 丙/辛日 | 戊子 | 丙辛从戊起 |
| 丁/壬日 | 庚子 | 丁壬庚子居 |
| 戊/癸日 | 壬子 | 戊癸起壬子 |

```
🔮 实战：时柱计算

function hourPillar(dayStem, hour) {
  // hour: 0-23
  const stems = ['甲','乙','丙','丁','戊','己','庚','辛','壬','癸'];
  const branches = ['子','丑','寅','卯','辰','巳','午','未','申','酉','戌','亥'];
  
  // 小时 → 时辰地支索引
  // 23:00-00:59=子(0), 01:00-02:59=丑(1), ...
  const branchIdx = hour === 23 ? 0 : Math.floor((hour + 1) / 2);
  
  // 日干决定子时天干起点
  const dayStemIdx = stems.indexOf(dayStem);
  const startStem = (dayStemIdx % 5) * 2;
  
  const hourStemIdx = (startStem + branchIdx) % 10;
  
  return stems[hourStemIdx] + branches[branchIdx];
}

// ⚠️ 子时问题：23:00-23:59 属于「夜子时」还是「早子时」？
// 不同流派有不同看法。建议在程序中提供选项让用户选择。
```

## 六、本课心法

> *"干支背熟了，各种关系理清了，你就有了一把万能钥匙。后面学什么术数，都是在这个框架里转。"——师父*

1. **天干主外，地支主内**：天干是明面上的，地支是暗藏的（地支还藏干）
2. **合冲有条件**：不是碰在一起就合/冲，要看位置和力量
3. **月令为王**：判断旺衰，先看月支（月令），再论其他
4. **查表不推算**：干支计算中，凡涉及历法的部分，一律查表
