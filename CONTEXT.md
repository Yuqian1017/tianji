# 天机卷领域术语

本文件只定义项目中的稳定术语。产品范围、里程碑和验收条件见 `docs/PRD.md`。

## 产品术语

| 术语 | 定义 | 不等同于 |
|---|---|---|
| 天机卷 | 以中国传统术数与中医知识为底座，结合教学、练习、实战工具与修仙式进度的统一产品 | 当前排盘 Web App |
| 实战工具层（Product A） | 输入信息后由本地引擎排盘或计算，再由 AI 提供解释的工具集合 | 完整教学游戏 |
| 修仙学堂（Product B） | 以课程、练习、测验、灵力、境界与渡劫构成核心循环的教学游戏 | 排盘引擎 |
| 统一产品（Product C） | Product A 与 Product B 共享账户、知识底座、进度和练习事件后的最终形态 | 把两个旧页面简单拼接 |
| 知识域 | 可独立组织课程、规则、术语和数据的主题范围，如六爻、八字、中医 | 前端 Tab |
| 课程单元 | 有明确学习目标、知识点、例题、练习和掌握条件的最小教学单元 | 一整篇 Markdown |
| 实战事件 | 用户完成一次排盘、查询、辨识或案例分析后产生的可记录学习行为 | AI 对话本身 |

## 数据术语

| 术语 | 定义 | 不等同于 |
|---|---|---|
| 原始来源（raw source） | 按原貌保存的 compendium、Skill、教材摘编或项目静态表 | 已验证数据库 |
| 候选条目 | 从原始来源识别出的、值得吸收或用于校正的知识记录 | 已接受事实 |
| 规范化条目 | 已按稳定 schema 拆出 ID、领域、正文、来源、安全、版本和关系字段的记录 | 已校勘条目 |
| 已校勘条目 | 已按该领域预定校验规则完成核对，并记录校验人、依据和日期的规范化条目 | 仅通过格式校验的条目 |
| 来源链（provenance） | 一个条目从原书/资料到整理稿、规范化记录和产品消费点的可追踪关系 | 只写一个文件路径 |
| 覆盖状态 | 分别记录“来源存在、已规范化、已校勘、已接入、已有教学内容” | 单一完成百分比 |
| 安全元数据 | 中医条目中的适用级别、禁忌、毒性、剂量来源、红线征象和就医升级规则 | 页面底部免责声明 |

## 边界术语

| 术语 | 定义 |
|---|---|
| 传统文化解释 | 对术数规则、历史体系和文化语境的教学或娱乐性解释，不作为事实预测承诺。 |
| 中医学习参考 | 对中医理论、辨证、方药、针灸与经典的教育性资料，不替代执业医师诊疗。 |
| 术数交叉健康 | 如“八字看健康”等传统术数内容，必须与中医知识条目分域、分证据等级展示。 |
