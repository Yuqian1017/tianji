## 2. 已修复的漏检与误导

### 两支相刑漏检

旧引擎只在寅巳申或丑戌未三支全齐时输出三刑，因此会漏掉“寅刑巳”“巳刑申”“丑刑戌”等两支关系。《三命通会·论三刑》逐项列出这些方向，三支齐全才适合汇总为一组三刑。

修复后：

- 两支出现时输出具体相刑方向。
- 三支齐全时只输出一个“三刑”，不重复输出三个两支结果。
- 子卯相刑和辰午酉亥自刑继续按各自规则检测。

### 三合、三会误写成已成化

旧标签直接写“申子辰合水”“寅卯辰会木”。三支出现只能确认关系，不足以证明已经成化。《三命通会》关于合化成局还要求旺气、月气、无伤克等条件，并明确干合不成局时不得论化。[合化成局](https://www.shidianguji.com/book/SK1610/chapter/1kf5v7gfo2iwj)

修复后标签只显示“申子辰三合”“寅卯辰三会”，目标五行放在 `huaCandidates`，状态固定为 `not_evaluated`。

### UI 不再暗示吉凶

旧 UI 用绿色表示合会、红色表示冲刑害，容易把关系直接解释成吉凶。《三命通会·论冲击》和《论三刑》都明确说明同一关系可能因命局条件而吉或凶，不能概括判定。

修复后所有关系使用中性色，并明确显示“仅列查表关系，未判断合化或吉凶”。

Playwright 使用 `1993-12-08` 酉时样例完成真实页面复验：桌面与 `390x844` 移动视口均显示结构化神煞及起查基准，关系区无截断、溢出或吉凶色彩暗示。

## 3. 相破保持 inactive

项目数据中存在六破候选表，但旧引擎没有消费。来源之间存在实质差异：较早资料只列子酉、卯午、丑辰、未戌，并可能排除寅亥与巳申；后期六破表加入这两组。

本轮不把数据存在误报为规则已通过：

- `poStatus = school_difference_inactive`。
- Runtime 不输出相破。
- 后续只有在明确采用某个版本并记录来源后才能启用。

## 4. 神煞边界

桃花、驿马、华盖的 36 个映射与传统三合分组一致：[咸池](https://zh.wikisource.org/wiki/%E4%B8%89%E5%91%BD%E9%80%9A%E6%9C%83/%E5%8D%B7%E4%BA%8C)、[驿马](https://www.shidianguji.com/zh/book/SK1610/chapter/1kt6e3p8yn2uf)。

但是：

- 年支、日支并用属于项目选择的流派口径，不代表唯一查法。
- 原典中的人格、性别、疾病和吉凶断语没有获得现代事实证据。
- Runtime 因此只返回结构化 `lookup_only` 标签、目标支和起查基准。
- UI 与 AI 输入明确标注“不作吉凶”。
