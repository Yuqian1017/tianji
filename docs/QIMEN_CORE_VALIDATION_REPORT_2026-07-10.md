## 1. 结论

修复后，奇门工具壳层和重新实现的确定性核心可以继续复用，不需要放弃整个项目重搭。

项目内审计覆盖 2020-2024 共 23,751 张盘、1,900,230 项检查，结果 0 fail。隔离安装的 `3meta@2.6.0` 对照相同 23,751 张盘、1,377,558 个字段，奇门核心 0 mismatch。`kinqimen@0.3.1` 对照 2024 年 4,758 张盘的共享口径，189,789 个字段 0 mismatch。

## 2. 修复的问题

### F-QM-001 节气整日提前切换

旧代码只传日期，交节日从 00:00 整天切换新节气，并保留固定月日近似回退。现改为公历年月日时分精确查当前节气；2024 芒种和夏至前后分钟已成为回归。

### F-QM-002 三元公式错误

旧代码从节气后第一个甲/己日开始按天数切三段。现按《奇门遁甲统宗》《奇门法窍》的符头支规则，向前取最近甲/己日，并按子午卯酉、寅申巳亥、辰戌丑未定上中下元。

### F-QM-003 地盘系统性错误

旧代码沿外围八宫“洛书轨迹”排三奇六仪，并强制最后的乙入中五。现按九宫数字序列布地盘：阳遁顺排、阴遁逆排，中五正常参与。18 种阴阳局均已枚举。

### F-QM-004 六甲遁干错误

旧代码把所有甲时都当作甲子戊。现从完整时干支取旬首，分别处理甲子戊、甲戌己、甲申庚、甲午辛、甲辰壬、甲寅癸。

### F-QM-005 值使与八门错误

旧代码让值使门直接落到时干地盘宫。现从值使原位按旬内步数在九宫数字序列中顺逆数；中五原位先寄坤二，落中时阳遁寄艮八、阴遁寄坤二，再沿八宫几何顺序布门。

### F-QM-006 四盘寄宫和顺序混杂

旧实现把天禽只留中五，并混用地盘绕行和几何转盘顺序。现九星、天盘干、八门和八神分别使用明确锚点；天禽保留中五身份并随天芮同宫。

### F-QM-007 解释层越界

旧 Prompt 和快捷入口要求求财、疾病、诉讼胜负和吉方吉时。现 UI、结果对象和 AI 上下文区分 `validated_declared_school` 与 `not_validated`；高风险建议入口已移除，未经验证的星门吉凶色和“官灾/大凶”等断语不再进入运行时显示或 AI 上下文。

## 3. 独立证据差异

`3meta@2.6.0` 的奇门核心完全一致，但它在立春公历日整天提前切年柱，产生 38 个 year-only 差异。项目沿用此前已验证的精确立春时刻口径；年柱差异不影响本轮奇门核心比较，但已原样保留在 artifact。

`kinqimen@0.3.1` 的地盘、天盘干、九星和八神等共享字段一致；它的值使门使用时干索引表，并保留不同的中宫落点规则，因此所有值使落宫与八门字段被明确排除。该包只能作为共享口径第三证据，不能裁决本项目的八门算法。
