## 5. 技术备忘（坑与契约）

- **Push 大包会断**（RPC failed / SSL）：`git rev-list --reverse origin/master..HEAD | awk 'NR % 15 == 0'` 取分段点逐段 `git push origin <hash>:master`，中断段会被后续段祖先链覆盖。
- resumeCast 契约不变；teachMoment/dressingUpdate/favorBranch 均静默自结算，勿改停留。
- 掌握 +5 统一 CP reward；各章对账 37/10；ch3 lint 的 favor 语义=**optimal-path max**（fail 专属链 BFS 差集排除）。
- 动态文本：`natalPalaceText()`（state.js，节点/选项标 `dynamicNatal`）。
- ch4 预判契约（数据化时评估，勿提前建）：castInteraction `fixedDate`（剧情占日恒定防旺衰漂移）；DressingBoard 旺衰标注位。
- NPC 立绘注册 `NPC_PORTRAITS`（presentation.js）；新 NPC 走 `scripts/generate-npc-portraits.py` 管线（gpt-image-2 → fal birefnet → webp）。
- 转录管线：双 agent 并行（幕段拆分+固定锚 id：如 `ch3-s5-300` 集市锚）→ 合并壳文件 → `audit:chapterN` 首跑——ch2/ch3 两轮均一次全绿。agent spec 要点：verbatim 铁律 + 节点契约清单 + 逐项自查（node --check/保真抽查/账目计数/图遍历）+「不确定处明报勿自作主张」。
- witness 提取口径：去 script/style + 去标签 + 逐行 strip 的行号（`/tmp/ctext-*.txt` 可随时从 `database/sources/ctext/*.html` 重生成）。
