# 天机卷项目路线图

更新日期：2026-07-10

## 当前结论

天机卷的统一产品由三个相互依赖、可以分阶段交付的部分组成：

- 实战工具层（Product A）：当前 repo 已有模块壳层和部分已验证的确定性核心，不能等同于完整 Product A。
- 修仙学堂（Product B）：课程、测验、掌握度、灵力、境界和渡劫。
- 统一产品（Product C）：让教学与实战共享知识底座、进度和用户记录。

Database 是三者共同底座，不单独作为面向用户的产品。

## 阶段

### V0 数据库与排盘正确性验证（当前）

目标：在继续产品实现前，重新证明当前排盘引擎、共享历法、静态表和中医高风险内容是否可靠。

当前顺序：

- 八字四柱、节气、日界、真太阳时与大运。
- 六爻、梅花、紫微、奇门、风水等确定性盘面。
- 基础查表、解释性知识和中医安全数据。

完成条件：当前产品全部 runtime 数据消费点进入 validation ledger；八字和共享基础取得独立复算结论；高风险路径被修复或隔离；其余 raw source 的全库验证继续作为并行 backlog，不永久阻塞已选教学切片。

V0 期间暂停教学/游戏实现和新的横向功能扩展。

### P0 主线恢复与数据库基线（待评审）

目标：结束“资料散落、产品主旨不清、数据库状态不可见”的状态。

完成条件：

- `docs/PRD.md` 成为主产品定义。
- `docs/DATABASE_COVERAGE_MATRIX.md` 能看出每个领域有什么、缺什么、下一步补什么。
- `docs/PROMINENT_SOURCE_MANIFEST.md` 能追踪主要来源、版本关系、许可和校验状态。
- 首个切片具备可消费的 canonical 数据、版本化 schema、来源/流派字段和查询合同。
- 用户确认第一条教学垂直切片。

### P1 教学核心可玩切片

目标：完成一个可玩的学习闭环，暂不铺满全部领域。

建议范围：阴阳五行与八卦基础，或六爻基础。包含课程、练习、规范答案、灵力、一次渡劫、进度保存，以及至少一个来自现有工具的结构化实战事件与复盘。

完成条件：新用户能从课程进入练习、渡劫和一次真实工具实战，系统能说明他学了什么、错在哪里、如何复盘、掌握度如何变化和解锁了什么。

### P2 实战工具接入学习循环

目标：把 P1 的单工具闭环扩展到更多已验证工具和跨领域学习路径。

完成条件：至少一个排盘工具能根据已学知识生成练习任务、记录实战、触发复盘，并更新领域掌握度。

### P3 多领域课程与数据库扩展

目标：按覆盖矩阵逐域扩展，不按目录数量追求“大而全”。

优先顺序由三项共同决定：玩家价值、已有工具复用度、来源与校勘准备度。

### P4 叙事、NPC 与长期养成

目标：在学习循环成立后加入角色关系、成就、每日任务、剧情和可选道侣线。

完成条件：这些机制能提升继续学习和回来练习的动机。只增加文本和数值不算完成。

## 反漂移规则

- 新增一个排盘或问诊模块，不自动等于推进教学游戏。
- 原始资料导入，不自动等于数据库完成。
- AI 生成一套题，不自动等于教学内容可用。
- 境界、灵力和 NPC 只有在驱动学习行为时才算游戏进展。
- 中医方药、穴位和身体不适内容必须先过安全与来源门槛。

## 下一决策

共享城市与民用时区当前 374 城市域已通过：374 条 GeoNames 原始行固定到规范核心，276 个中国城市额外通过 31 个省级行政区合同；73,414 日均时差、17,952 城市时点、10 个 DST fixture 和五条运行时消费路径共 132,836 项检查 0 fail。拉斯维加斯错误时区、海外固定标准经线、闰年分母和重复/跳过时刻处理已修复。宿主 tzdb 版本、任意坐标和重复时刻前/后实例选择仍是明确边界。

中医 100 行/101 味药典候选层已完成当前公开证据范围的全量审计：2,468 项 0 fail；79 味命中 2025 一部目录，22 味缺失或未裁决；2020 官方历史正文命中 79 味，其中 38 味有数量差异。该通过只授证来源、身份与阻断完整性，101 味剂量全部继续 blocked。

下一 validation 批次直接转入妊娠、十八反十九畏、马兜铃酸、31 条方剂警示与穴位外治安全层。当前产品中的中医药物、剂量、居家急救、裸艾灸和疾病推断继续 blocked，不因 Skill 内容较全而自动恢复。

已通过的确定性基础包括：八字四柱/节令/声明口径大运、六爻、梅花、共享《周易》、紫微、奇门、风水下卦、五运基础年结构、子午基础时辰对应和共享城市/时区当前域。各领域的吉凶、疾病、财富、婚姻、成败、方位和临床解释仍按 ledger 单独 blocked。

第二轮 PRD fresh review 为 HOLD。全库审计继续执行，但不再把无界 raw-source 验证写成教学产品的永久串行总闸；在当前产品风险闭环后，应另行确认“基础通识 + 最小实战事件”或“六爻入门”首切片，并补 canonical 数据合同、流派/口径字段和掌握度算法。
