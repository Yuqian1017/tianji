# 天机卷 Database Coverage Matrix

- 文档状态：基线盘点
- 版本：0.1
- 更新日期：2026-07-10
- 范围：项目内 raw database、当前运行时代码、教学/游戏所需数据
- 配套文档：`PRD.md`、`PROMINENT_SOURCE_MANIFEST.md`

## 1. 结论

现有 compendium 和中医 Skill 都有大量内容可以补充进天机卷 Database。

项目已经找回了主要资料。现阶段缺少统一 schema、逐条来源链和校勘状态；内容仍分散在 raw Markdown、Skill references、原文文本和功能内静态 JS 中。

可吸收范围包括：

- compendium：基础宇宙观、易经与 64 卦、术数规则、案例、口诀、查表、姓名、择日、六壬、相术、教学难度和游戏设计素材。
- 中医 Skill：基础理论、四诊、辨证、50 余内科病种、中药、方剂、针灸、经典、医家、温病和安全约束。
- 当前 app：已运行的算法、静态表、提示词、用户流程和历史数据结构，可反向作为“现状来源”纳入数据库审计。

所有内容都可以先进入 raw/reference 层；是否进入 normalized、accepted 或 product-eligible 状态，分别由结构化、校勘、安全和许可门槛决定。

## 3. 状态口径

矩阵不使用单一“完成率”，而是分别记录以下状态：

| 维度 | 含义 |
|---|---|
| Raw | 项目内是否保存了原始资料或静态表 |
| Normalized | 是否已拆成稳定 schema 的条目 |
| Reviewed | 是否按该领域规则做过内容或算法校核 |
| Runtime | 当前产品是否已经消费该内容 |
| Learning | 是否已有课程、练习、规范答案和掌握度映射 |
| Product eligible | 许可、安全、来源和校勘是否允许进入目标产品场景 |

状态值：`无`、`少量`、`部分`、`较全`、`已实现`。其中“较全”只表示相对覆盖面，不表示权威或无误。

## 5. 可以从 compendium 补什么

### 5.1 可直接进入候选池

这些内容适合先拆成候选条目，不要求先重写：

- 基础实体：天干、地支、五行、八卦、六十四卦、星曜、宫位、九星、八门、二十四山等。
- 关系与查表：生克、合冲刑害、纳甲、世应、六亲、十神、四化、飞星、节气、长生、神煞等。
- 教学材料：章节、知识点、难度标签、口诀、案例和术语。
- 算法规则：起卦、装卦、排盘、查询与评分步骤。
- 未实现工具：64 卦查询、姓名五格、择日、六壬草稿。
- 相术补充：面相与手相条目，可与当前 ML 特征和问卷字段建立映射。

### 5.2 适合用于校正当前 Runtime

- 比较当前 JS 静态表与 compendium 查表是否一致。
- 把当前引擎中的派生常量回连到来源文件。
- 用 `reference/test-cases.md` 和当前测试构建逐域回归用例。
- 把 Prompt 中的规则句拆回可追踪知识条目，避免 Prompt 成为隐形数据库。
- 找出当前已实现工具中被简化、扩展或改写但没有文档记录的部分。

### 5.3 暂不能直接标为 accepted 的内容

- 文件自己标注“简化”“近似”“仅供参考”或“不可用于生产”的算法。
- 没有来源字段的断语、口诀和吉凶表。
- 康熙字典笔画、择日禁忌和六壬完整性等需要外部数据或专门校核的内容。
- 从传统体系推导到现代健康、财富或人生结果的确定性表述。

这些内容仍可进入候选池，只是状态应保持 raw/normalized-unreviewed。

## 7. 横向数据类型覆盖

| 数据类型 | Raw | Normalized | Runtime | 主要缺口 |
|---|---|---|---|---|
| `source_document` 来源文档 | 有 | 无 | 不适用 | 稳定 source_id、版本、许可、实际文件统计 |
| `knowledge_item` 知识条目 | 有大量正文 | 无 | 静态表局部使用 | 稳定 ID、类型、难度、关系、状态 |
| `entity` 基础实体 | 有查表 | 无统一模型 | 各模块重复定义 | 去重、别名、跨域引用 |
| `rule` 规则 | 有 | 无 | 引擎和 Prompt 内重复 | 适用条件、例外、算法版本、来源 |
| `algorithm_case` 算法用例 | 有少量 | 无 | 各模块自有测试 | 统一 fixture、预期结果、证据等级 |
| `interpretation_case` 解读案例 | 有 | 无 | Prompt 局部使用 | 输入、步骤、结论、适用边界 |
| `lesson` 课程单元 | 有难度映射 | 无 | 无 | 学习目标、前置、知识点、示例、掌握条件 |
| `assessment_item` 题目 | 只有 Prompt 草稿 | 无 | 无 | 规范答案、解释、来源、版本、质量状态 |
| `safety_rule` 安全规则 | TCM Skill 有 | 法定毒性 28 项 + 药典/妊娠方剂/穴位外治/病种红线/理论诊断五个候选层 | 高风险 runtime 已隔离 | 五个候选层仍 blocked；继续完整中药/方剂、现行药典、逐穴定位和食疗养生验证 |
| `runtime_mapping` 运行时映射 | 无统一登记 | TCM 首批 legacy baseline | 13 模块 | 继续扩展到其他领域；TCM 旧消费点已有可逆清单 |

## 9. 覆盖完成判定

一个知识域只有同时满足以下条件，才可标记“可用于教学/产品”：

1. 主要 raw source 已登记，版本和权利状态清楚。
2. 核心实体、关系、规则和案例已规范化。
3. 关键冲突和算法用例已记录，校勘状态可见。
4. 课程和计分题都回指 accepted 知识条目。
5. Runtime 消费点有映射和回归验证。
6. 中医等高风险域通过额外安全门槛。
