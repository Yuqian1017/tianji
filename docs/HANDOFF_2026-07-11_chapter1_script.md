## 4. 创作素材存量（上一 session 的构思，可用可改）

### 剧情梗概（PRD § 9.1 已定框架）

玩家入门术数宗门 → 引路人带新人 → **门中失窃事件** → 引路人带玩家摇人生第一卦 → 解卦教学 → 玩家凭卦象在两条线索中抉择 → 真相揭晓 + 好感萌芽 + 宗门悬念。

### 引路人角色基调（可细化，勿推翻）

- 性格：**外冷内热，毒舌带你飞**。教学时刻的性格载体——毒舌给纠错提供叙事糖衣（说错了她怼你，但怼完把正确的讲透）。
- 知识域：六爻。世界观口头禅方向："物有其时，卦有其应"。
- 好感萌芽要**微**：章末只是语气软了半分/多看了你一眼，不要表白级推进。

### 建议分幕（参考，可自由重排）

1. 开场：入门（世界观最小交代，玩家性别/称谓选择自然嵌入）
2. 遇见引路人（性格立住，毒舌第一击）
3. 失窃事件爆发（剧情张力起点）
4. 示范时刻：引路人当众解一卦（玩家看，KP-001 见过）
5. 引导时刻：带玩家亲手摇卦（三钱六掷交互演出，KP-002 用过；装卦认爻，KP-001 用过）
6. 教学时刻：解这一卦，讲动爻（KP-003 见过→引导）
7. **独立抉择点 1**：凭卦象选追查方向（KP-003 独立，主计分点）
8. 追查段落（可埋第二个小抉择点：KP-001 独立——比如认出某处标记是阳爻还是阴爻）
9. 真相 + 好感时刻 + 悬念钩子

### 摇卦交互的演出要求

三枚铜钱、六次投掷、逐爻显示——这是**真实六爻起卦**（引擎已验证），剧本里写清楚每一掷之间引路人的反应台词位（M1 时交互组件按此实现）。剧情需要特定卦象来推动（比如需要一个有动爻的卦），剧本里注明"此处卦象剧情固定为 X 卦动第 N 爻"——首章卦象**剧情预设**而非真随机（保证教学与剧情可控），但摇卦手感真实。这个设计取舍已定，写进剧本注释即可。

### 抉择点设计心法

- optimal 的判定依据必须是**本章教过的知识**能推出来的，不许要求超纲推理。
- wrong 不惩罚：走平淡线（少一段剧情/少一点好感），并触发 KP 待复习标记。选错的玩家在真相揭晓时会自然明白"哦，卦早就说了"——这本身就是复习时刻。
