# 天机卷主 PRD Fresh Review

- 日期：2026-07-09
- 审查对象：`docs/PRD.md` v0.1
- Reviewer：独立只读 fresh reviewer；主线程逐项复核证据
- 结论：`HOLD`
- 边界：本审查判断 PRD 能否指导下一阶段，不证明任何术数或中医数据正确

## 1. 总结

主线方向成立：天机卷应是共享 Database 支撑的教学、练习、游戏进度和实战工具系统，当前项目不需要整体推倒重搭。

但原稿把“可复用的产品壳层”和“可信的领域引擎/内容”混在一起。现阶段可以保留前端框架、交互、账户、历史记录和 AI 管道；八字、六爻、紫微、奇门等算法与静态表，以及中医具体建议，必须逐域通过来源映射、fixture、独立复算和安全审查后，才能成为课程规范答案、计分依据或学习事件来源。

因此 PRD 暂不进入教学实现，先完成当前 V0 correctness validation。

## 2. 已验证阻塞项

### PRD-R01 P0：当前 Product A 中医安全没有进入 P0 验收

证据：

- PRD 已规定方药、剂量、针刺和艾灸不能作为无门槛奖励，并要求具体安全字段。
- 当前体质模块直接包含黄芪 15g、肉桂 3g 等剂量、多个穴位及“艾灸为主”等操作建议。
- 八字健康 Prompt 要求 AI 给出食疗、茶饮、忌口、穴位和健康风险。
- 原 P0 完成条件只要求三份产品/数据库文档一致，没有要求先盘点或隔离这些当前运行时消费点。

决定：把“当前 Product A 中医具体消费点全量 inventory、风险分级、未审内容隔离”列为 V0/P0 gate。免责声明不能代替逐条安全规则。

### PRD-R02 P0：未验证引擎不能作为可信教学基础

证据：

- Coverage Matrix 明确 normalized database 为 0，各领域只做过“部分/少量”复核。
- PRD 又把当前工具定义为实践和验证环节，并在原重搭决策中默认保留排盘引擎。
- 当前八字首轮代码审查已发现节气覆盖、起运搜索、时间口径和启发式评分等高风险证据缺口；是否构成具体错误仍由 validation ledger 复算。

决定：保留实现基线，不预先承诺保留领域引擎。每个引擎和静态表必须通过该领域 gate，才可产生规范答案、计分、掌握度或渡劫结果。

## 3. P1 修订项

### PRD-R03：首个切片应证明课程到工具的闭环

“练气期基础通识”可以验证课程、练习和进度，但原范围只提供 64 卦/六爻预览，不能证明“课程 -> 工具 -> 判断 -> 复盘 -> 掌握度”的完整产品主张。

建议：保留基础通识作为前置，同时加入一个最小、已校验的 64 卦或六爻实战事件；若八字先完成验证，也可改为八字中的单一基础判断，但不能接入未通过 gate 的完整解读。

### PRD-R04：补 Product C 身份、事件和数据合同

PRD 目前要求共享账户、历史、学习状态和实战事件，但没有定义：

- 用户、匿名档案和道号之间的身份关系。
- 课程事件、答题事件、实战事件的稳定 ID、版本和幂等规则。
- 当前 SQLite/服务端数据与旧 localStorage 资产的迁移边界。
- 知识条目版本变化后，旧答题和旧盘面如何追溯。

这些合同必须在教学切片编码前确定，否则 Product A 和 Product B 仍会形成两套进度事实。

### PRD-R05：显式废止旧 AI 实时出题/评分规格

旧 `APP-SPEC.md` 写有“AI 出题 -> 用户作答 -> AI 评判”，旧 `APP-SPEC-B.md` 还要求渡劫题目实时生成且不需要题库。新 PRD 已实质纠正该方向，但尚未声明旧条款被 superseded。

决定：AI 可以生成草稿和表述变体；所有计分题必须绑定已审知识条目、规范答案、解释、版本和校验状态。旧规格中相冲突条款不再有效。

## 4. P2 修订项

### PRD-R06：权利不明内容在产品资格确认前仅限内部使用

PRD 已要求发布前确认许可，但对 Product C 开发期间的默认状态不够明确。建议所有许可未知、非商业或来源权利不明的内容标记为 `internal_only`，不得默认进入对外内容包、用户可见课程或 AI 可引用上下文。

## 5. 允许继续的部分

- 继续使用当前 repo，不整体重搭。
- 继续建设统一 provenance、schema、validation ledger 和 source manifest。
- 继续保留 UI shell、账户/历史、AI Provider、相机与通用交互作为候选基础设施。
- 继续把中医 Skill 和 compendium 吸收到 raw/reference 层；进入 normalized/accepted/product-eligible 仍需分别过门槛。

## 6. 解除 HOLD 的条件

1. 八字首轮 30 例和共享历法口径完成独立复算，形成保留/修复/替换结论。
2. 当前 Product A 中医具体药味、剂量、穴位、艾灸和健康风险输出完成 inventory；未达安全等级的路径被隔离。
3. PRD 补入 Product C 身份/事件/版本合同，并明确旧 AI 出题条款失效。
4. 首个垂直切片包含一个已校验的课程到工具闭环。
5. 对外可见来源默认满足 product-eligible；其余标记 `internal_only`。

解除 HOLD 不代表全数据库已经验证，只代表 PRD 可以安全地指导下一阶段实现。
