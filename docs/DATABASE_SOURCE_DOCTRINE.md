# 天机卷 Database Source Doctrine

- 状态：canonical
- 生效日期：2026-07-10
- 决策：典籍见证优先，现代边界分层附加

## 1. 目的

天机卷是传统文化学习系统。数据库的首要任务是忠实保存和整理传统知识，而不是先用现代监管、科学或产品风险标准筛掉古籍中真实存在的说法。

因此，面相、手相、术数和中医条目即使包含现代人不接受、无法验证或不宜直接实践的判断，只要确实见于可定位文献，就应保存在历史知识层。现代证据只决定如何标注和如何使用，不抹除其文献存在。

## 2. 四层模型

1. **典籍见证层**：作品、版本、影印/转录见证、篇卷页行、原文、异文和归属争议。
2. **传统解释层**：从原文归一化出的术语、规则和传统判断；必须逐条回指典籍见证。
3. **现代边界层**：科学证据、现行药典、监管、安全、伦理和现实决策限制；不得冒充传统内容的来源。
4. **产品呈现层**：决定条目可否用于文化讲解、计分题、排盘解释或现实行动；必须同时遵守来源和场景门槛。

## 3. 条目准入

任何进入 normalized/accepted/runtime 的传统知识条目至少需要：

- 稳定 `id`；
- `source_type`；
- 作品和版本见证；
- 可复核 `locator`；
- `original_text`；
- 与原文分开的 `normalized_claim`；
- `source_ref_ids`；
- `attribution_status` 与 `witness_status`；
- `interpretation_status`；
- `runtime_policy`。

找不到出处的内容不删除，可保留在 raw/candidate 层并标 `source_not_yet_located`。与已定位原文冲突的现代扩写必须显式记录，不能覆盖原文。

## 4. 典籍与现代资料的角色

- 《周易》、历代术数典籍、医经、本草、方书和医家著作负责证明传统文本“记载了什么”。
- 影印本优先作为版本见证；可搜索转录用于定位，关键条目后续与影印页复核。
- 现代教材可证明当代传承整理，但不能自动替代更早原典。
- 国家卫健委、药监、药典和现代临床资料只负责现行身份、剂量、安全、适应证、禁忌和现实使用边界。
- 现代资料不能证明或否定某段传统记载是否存在，也不能因为某说法不宜实践就从文化数据库删除。

## 5. 展示规则

- 有出处的传统判断可展示为“某典籍某版本记载/传统说法”，并附具体出处。
- 不把传统判断改写成现代科学事实；也不把“未被现代验证”改写成“古籍不存在”。
- 医疗、财务、雇佣、法律等现实决策继续使用更严格的产品边界。
- 同一主题的异文、流派差异和作者自我保留意见并列保存。

## 6. 首个落地样板

`database/xiangshu/` 已按本原则建立相术样板。首批核对显示：

- 《神相全编》确有“三停平等，一生衣禄无亏”等传统判断；
- 旧 compendium 的“上停长：少年得志、学业佳”等文字与当前定位见证“上停长少年忙”不一致；
- 《公笃相法》支持五形手的形态分类，但作者在同段自陈手掌相法经验甚少；
- 《许负相法》序言承认文本含后世增饰，传统归属与现存文本层次必须分开记录。

该样板验证的是来源忠实性，不证明传统判断具有现代预测效力。
