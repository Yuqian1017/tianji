## 3. 已修复问题

### 乾坤外卦纳干错误

旧 `NAJIA` 只有一个 `stem` 字段，导致乾作外卦仍纳甲、坤作外卦仍纳乙。项目自己的伏神表已使用正确的乾外壬、坤外癸，因此 runtime 和 database 内部互相矛盾。

现改为 `innerStem` / `outerStem`：乾内甲外壬，坤内乙外癸，其余六卦内外同干。来源锚点为[《卜筮正宗·纳甲装卦歌》](https://ctext.org/wiki.pl?chapter=889452&if=gb&remap=gb)。

### 月建与年建使用固定日期近似

旧六爻引擎固定以 2 月 4 日换年，并用每月 4-8 日的近似日期换月建，真实节令时刻附近会排错。

现复用已通过八字独立验证的 `createBaziCalendar()`，2026-02-04 03:55 与 04:03 分别正确落在乙巳年己丑月和丙午年庚寅月。

### 非法爻值被静默接受

旧引擎只检查数组长度，0、5、10 等值也会按奇偶生成卦。现只接受三钱法声明的 6、7、8、9。

## 4. 伏神边界

[《卜筮正宗·伏神正传》](https://ctext.org/wiki.pl?chapter=801184&if=gb&remap=gb)明确：本卦缺所需六亲时，才从本宫首卦对应爻取伏神。

- 当前 runtime 只列本卦缺失的六亲，语义符合本轮声明口径。
- 64 卦全量检查未出现一个缺失六亲对应两个本宫候选的歧义。
- 飞伏生克与后续吉凶判断不随伏神位置表通过而自动授证。

## 5. 解释层边界

旧 AI prompt 把“六冲事散、六合事成、世空心不诚”等写成无条件结论。结构排盘一致不能证明这些现实预测。

现已：

- 设置 `chartStatus = validated_deterministic`。
- 设置 `interpretationStatus = not_validated`。
- 要求 AI 区分结构观察、传统解释与现实边界。
- 禁止把卦象包装成事实预测、医疗诊断、法律判断或投资建议。
- UI 明示“结构已校验，现实预测与吉凶解释未校勘”。

Playwright 以全六老阳固定样例验证真实页面：本卦乾为天正确显示外卦 `壬午/壬申/壬戌`，变卦坤为地显示外卦 `癸丑/癸亥/癸酉`；桌面与 `390x844` 移动视口均无截断或溢出，解释边界可见。

## 6. 当前复用边界

可以复用：京房八宫卦宫、世应、纳甲、六亲、六神、旬空、伏神位置、动爻与变卦结构。

仍然 blocked：

- AI 或课程把传统断语当作可验证的现实答案。
