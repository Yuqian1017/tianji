## 1. 一句话定义

**一个玩完之后你真的会算卦的修仙恋爱剧情游戏。**

展开版：以权威典籍校准的东方玄学知识为世界物理法则、以章节制剧情推进、以"角色即课程"承载教学的修仙恋爱游戏。玩家在追剧情、攻略角色的过程中，不知不觉学会真实的术数——因为剧情里的每一卦都是真卦，每个盘都是真盘。

## 2. 设计哲学 — 四支柱

### 支柱一：术数即游戏机制（不做"游戏化的课程"）

玄学不需要游戏化，它本来就是游戏机制：

| 术数 | 天生的游戏系统 |
|------|----------------|
| 六爻摇卦 | 随机事件 + 决策系统 |
| 八字 | 角色生成器 + 人物内在逻辑 |
| 五行生克 | 相性/克制系统 |
| 风水择日 | 空间策略 + 时机策略 |
| 奇门 | 战术布局系统 |

**玩的过程就是在用知识**：剧情拉动学习（pull），不设做题门槛阻挡进度（push）。玩家因为想知道"这一卦到底说了什么"而学，不因为"不通过渡劫不给玩"而学。

### 支柱二：角色即课程，恋爱即教学

每个可攻略角色绑定一个知识域。她/他教你她的领域，因为剧情里她就靠这个立身。好感线与学习线是**同一个系统**：想读懂角色的心结，你得看懂她的八字；想给心上人挑对生辰礼物，你得会看喜用神。攻略她 = 学会她的领域。

### 支柱三：权威数据库是底色（知识真实性不可妥协）

市面上修仙游戏无数，但"里面每一卦都是真卦、玩完你真的会算"的游戏是零——这是本产品唯一护城河。数据库不是玩家翻阅的百科，是**这个世界的物理法则**：

- 进入剧情的每个知识事实，必须能回溯到权威典籍的具体出处（书名 + 版本 + 定位）。
- 排盘由已验证的确定性引擎计算；AI 只解释，不算卦。
- 编剧从数据库取事实，不从记忆取。剧本中每个知识引用带 `source_ref`。

### 支柱四：寓教于乐，节奏有呼吸

教学章节与纯情感/纯游戏性的 bonus 章节交替，学习密度有起伏。教学严谨性藏在骨架里，玩家感受到的永远是故事。**严谨的骨架，叙事的皮肤。**

## 3. 产品形态 — 章节制剧情游戏

```
序章 → 主线章1 → 主线章2 → [bonus 章] → 主线章3 → 渡劫章 → 主线章4 → ...
        (教学)    (教学)    (情感/游戏)   (教学)    (综合案例)
```

两类章节：

| 章节类型 | 教学负担 | 作用 | 产出数值 |
|----------|----------|------|----------|
| **主线章**（教学章） | 每章新知识点 ≤3 | 推进世界观主线 + 教学时刻 + 隐形评估 | 灵力 + 掌握度 + 好感 |
| **Bonus 章** | 零新知识 | 感情浓度高的角色专属剧情 / 纯游戏性玩法（斗法、探秘、节庆）；节奏呼吸口 | 好感为主 + 少量灵力 |

Bonus 章中知识只以"彩蛋"形式软复现（如：她生辰宴上你送出喜用神颜色的礼物 → 额外好感），不教新知识、不计掌握度。

单章体量：主线章 30–60 分钟；bonus 章 15–30 分钟。平台：Web（复用现有 React + Vite 壳）。

## 4. 教学体系设计（核心章节）

> 设计目标：教学论上站得住（情境学习 + 掌握学习 + 间隔重复 + 费曼式输出），但玩家全程感受不到"课程"的存在。

### 4.1 知识点模型 — 最小教学单元

知识点（Knowledge Point, KP）是教学、评估、数值三个系统共同的原子。每个 KP 必须具备：

| 字段 | 说明 | 示例（KP-LY-003 动爻） |
|------|------|------------------------|
| `id` | 稳定 ID：`KP-{域}-{序号}` | `KP-LY-003` |
| `domain` | 知识域 | 六爻 |
| `name` | 名称 | 动爻与变卦 |
| `canonical_text` | 典籍原文（照录，不改写） | 《增删卜易》相关章句 |
| `source_ref` | 书名 + 版本 + 定位（章节/页/URL） | 增删卜易·野鹤老人，ctext 定位 |
| `modern_explain` | 白话解释（标注为整理者转述） | "老阳老阴为动，动则变出新卦" |
| `game_line` | 游戏内 NPC 教学口径（≤2 句，剧本可直接用） | 师姐台词版本 |
| `difficulty` | 必修/进阶/专研（沿用 DIFFICULTY-MAP 分级） | 必修 |
| `prereq_ids` | 前置 KP | KP-LY-001, KP-LY-002 |
| `status` | raw → normalized → reviewed → **accepted** | 进剧情必须 accepted |
| `safety_level` | 中医域必填（沿用旧 SAFE 条款） | n/a |

**硬规则**：`status ≠ accepted` 的 KP 不得进入剧本教学时刻或抉择点判定。原文与转述分离（继承 DB-008）。

### 4.2 掌握度 — 五阶状态机（剧情行为驱动，不靠做题）

```
未接触 → 见过 → 用过 → 掌握 → 内化
        (示范)  (引导)  (独立)  (跨章复用)
```

| 状态 | 达成条件 | 灵力奖励 |
|------|----------|----------|
| 见过 | 剧情中 NPC 示范该 KP | +1 |
| 用过 | 引导应用完成（NPC 在旁提示下使用） | +3 |
| 掌握 | 独立应用成功（无提示抉择点选中 optimal） | +5 |
| 内化 | 跨章节回响事件中再次独立应用成功 | +8 |

**掉级规则**：独立应用失败 → 该 KP 标记"待复习"，系统在后续剧情中自动安排复习时刻（NPC 换个场景侧面再讲，或 bonus 对话软复习）。失败永不清进度、不锁主线——只走平淡剧情线（继承 ASM-004 精神）。

### 4.3 教学时刻类型学 — 知识进入剧情的五种方式

每个主线章由这五类"教学时刻"编织，剧本中每个教学时刻显式标注类型与 KP：

| # | 类型 | 形态 | 教学论依据 | 第一章示例 |
|---|------|------|------------|-----------|
| 1 | **示范**（Demo） | NPC 做，你看 | 观察学习 | 师姐当众解卦 |
| 2 | **引导应用**（Guided） | 你做，NPC 提示 | 脚手架 | 师姐带你摇卦、认爻 |
| 3 | **独立应用**（Independent） | 抉择点，无提示 | 掌握学习 | 凭卦象选追查方向 |
| 4 | **教授**（Teach-back） | 你教剧情中的他人 | 费曼技巧 | 后续章：向新入门弟子解释动爻 |
| 5 | **回响**（Recall） | 跨章自然复用旧 KP | 间隔重复 | 第三章剧情再次需要第一章的动爻概念 |

### 4.4 认知负荷规则（章节设计硬约束）

- 每主线章**新 KP ≤ 3**。
- 每个新 KP 在本章内至少经历 示范 → 引导 → 独立 三个时刻（完整闭环不出章）。
- 新术语首次出现必须有剧情内解释；未教的术语可作氛围词出现，但不得成为抉择依据。
- 下一章开工前先列 KP 清单，超 3 个就砍章或拆章。

### 4.5 间隔复用规则（章节序列硬约束）

第 N 章的新 KP，必须在第 N+1 至 N+3 章中至少安排一次**回响时刻**（剧情自然需要，非复习题）。章节规划表中为每个 KP 维护回响排期——这是把间隔重复藏进章节设计的机制。

### 4.6 隐形评估 — 抉择点即测验

旧 PRD 的评估需求（ASM 系列）不作废，**变形**为剧情抉择点。剧本中每个计入掌握度的抉择点必须标注：

```yaml
choice_point:
  id: CH1-CP-02
  tests_kp: [KP-LY-003]          # 测的知识点
  options:
    - text: "跟着变卦的方向去后山"
      verdict: optimal            # 判定：optimal / suboptimal / wrong
      basis: "世爻动而化进…"       # 判定依据
      source_ref: 增删卜易·卷X     # 依据出处
    - text: "去前殿守着"
      verdict: suboptimal
  reward: { optimal: {灵力: +5, 好感: +2, 解锁: 暗线片段} }
  on_wrong: 平淡线 + KP 标记待复习   # 永不锁进度
```

**质量指标（继承旧 PRD，变形口径）**：计入掌握度的抉择点 100% 关联 KP、判定依据与 source_ref；剧本构建时跑一致性 lint（脚本校验每个抉择点标注完整、引用的 KP 均为 accepted）。

**AI 边界（继承 § 11）**：AI 可为抉择点生成台词变体、氛围文案，但 verdict 与 basis 来自数据库，AI 不得即兴决定对错。

### 4.7 实战证据闭环 — 记录观察，不把“应验”变成知识权威

实战工具可生成结构化练习记录，但“现实结果是否符合事前判断”与“玩家是否掌握 accepted KP”是两套独立证据。天机卷吸收 `xuanxue-database-skill` 的命例/断语/应事/方法关联结构，经严格化后采用以下五类记录：

| 实体 | 作用 | 硬约束 |
|---|---|---|
| `practice_case` | 固定一次命盘/卦盘与场景 | 保存引擎版本、流派口径、时间/时区精度和不可变盘面快照 |
| `method_reference` | 关联 KP、典籍来源与流派方法 | 典籍裁决状态与个人观察状态分字段，后者不得提升前者 |
| `prediction_claim` | 保存事前判断 | 结果窗口开始前冻结；时间窗、可观察标准和证伪条件必填，修正只追加不覆写 |
| `outcome_observation` / `adjudication` | 保存后来发生的事实与裁决 | 区分自述、同期记录、第三方和客观记录；未随访、未解决和排除项必须留在分母 |
| `learning_reflection` | 把实战转为复习行动 | 可安排回看来源、重排盘或再练习，但默认不直接增减掌握度 |

个人记录只可展示为“用户观察到的结果摘要”，不得标为方法已被客观验证。医疗、法律、财务和其他高风险判断不进入应验计分。候选合同见 `database/learning/practice-evidence-schema-candidate.json`，完成产品、隐私与数据迁移评审前不进入 runtime。

## 5. 修仙体系融合 — 境界即学习阶段

### 5.1 境界映射表（首期定前四境，旧 GAME-DESIGN 15 级保留为长期资产）

| 境界 | 学习阶段 | 必修 KP 范围 | 渡劫（升境考验） |
|------|----------|--------------|------------------|
| 凡人 | 序章 | — | 无 |
| 练气 | 基础通识：阴阳爻、卦的构成、摇卦法 | 第一章 KP 全部 ≥ 掌握 | 独立起一卦并说清卦象结构 |
| 筑基 | 六爻入门：装卦、用神、旺衰初步 | 前三章 KP ≥ 掌握 | 独立完成一个完整占例初判 |
| 金丹 | 六爻进阶 + 第二域（八字）开门 | 六爻主干 + 八字基础 | 跨域综合案例 |

### 5.2 渡劫 = 综合剧情案例（不是考试）

渡劫是一个**完整剧情事件**（宗门大比、外出历练、突发危机）：玩家从起卦到断卦全程独立完成，NPC 不提示——"雷劫"就是案例的复杂度本身。

- 判定依据：数据库中已 accepted 的案例规范解读（典籍占例优先，如《增删卜易》原书占例改编）。
- 通过：升境 + 灵力 +30 + 剧情高光时刻。
- 失败：不清进度、不掉境界（继承 ASM-004），NPC 以剧情方式补课（针对薄弱 KP 的复习支线），可再战。
- 升境三重检查（继承 GAME-002）：灵力达标 + 必修 KP 全部 ≥ 掌握 + 渡劫案例通过。

### 5.3 三数值分离（继承 GAME-004：防"等级高但不会"）

| 数值 | 含义 | 来源 | 存储 |
|------|------|------|------|
| 掌握度 | 每 KP 的真实学习状态 | 教学时刻 + 抉择点结果 | 按 KP 记录，是唯一学习事实 |
| 灵力 | 修为经验值（掌握度的叙事化投影） | § 4.2 奖励表 + 章节通关 +10 / bonus +5 / 渡劫 +30 | 全局累计 |
| 好感度 | 单角色关系进度 | 剧情选择 + 知识应用质量 + bonus 章 | 按角色记录 |

灵力只能由明确学习/剧情事件产生（继承 GAME-001），不设可刷的重复操作。

## 6. 角色系统 — 角色即课程

### 6.1 角色位与知识域绑定

| 角色位 | 知识域 | 剧情功能 | 首期状态 |
|--------|--------|----------|----------|
| 引路人（师姐/师兄） | 六爻 | 第一章起的主教学者 + 第一条感情线 | 第一章即登场，档案详设 |
| 命理者 | 八字 | 其八字即其心结——攻略过程 = 读懂 TA 的命盘 | 金丹阶段开门，剧本阶段展开 |
| 堪舆者 | 风水/奇门 | 空间与布局的战术型角色 | 后续 |

设计规则：**角色的知识域 = 角色的立身之本 = 角色的性格来源**。六爻者信"物有其时"，八字者困于"命有定数"——知识域塑造世界观冲突，也塑造感情线的核心张力。

### 6.2 NPC 性别可选（F6）

方案：**角色位双性别变体，骨架共享**。

- 开局玩家选择引路人性别（师姐/师兄），后续每个可攻略角色位同理；玩家自身性别与称谓亦可选。
- 角色档案 = 共享骨架（知识域、剧情功能、教学时刻、案例、好感节点、核心性格）+ 性别变体层（称谓、外貌描写、部分对话风味、少量专属片段）。
- 剧本文案用模板变量：`{{senior}}` → 师姐/师兄，`{{ta}}` → 她/他。
- 成本估算：写作 +25%（变体层），非 ×2（骨架全共享）。
- 硬规则：两个变体的**教学内容与抉择点判定完全一致**——性别只改变叙事风味，不改变知识与数值。

### 6.3 好感度与知识应用联动

- 抉择点选中 optimal 且该 KP 是该角色所教 → 好感额外 +（"她看你把她教的东西用得漂亮"）。
- 好感等级解锁角色**私教剧情**（更深知识的 bonus 教学章）与感情剧情。
- 好感不参与升境判定（数值分离），但影响剧情分支与结局。

## 7. Bonus 章节体系（F5）

| 类型 | 内容 | 教学负担 | 触发 |
|------|------|----------|------|
| 情感章 | 角色专属日常/约会/心结剧情，感情浓度高 | 零新知识；旧知识仅作彩蛋 | 好感达标解锁 |
| 游戏章 | 斗法小游戏、探秘、节庆活动 | 零新知识 | 主线节点解锁 |
| 私教章 | 角色一对一深教（好感解锁的进阶知识） | 有 KP，但走完整教学时刻闭环 | 高好感解锁 |

设计规则：

- 主线每 2–3 章之间至少安排 1 个 bonus 章位——学习密度的呼吸口，宗主定调"寓教于乐"的机制保障。
- 情感章/游戏章不计掌握度、不出现计分抉择点；知识彩蛋（送对喜用神颜色礼物 → 额外好感）只奖励、不惩罚、不评估。
- 私教章是唯一带教学的 bonus 类型，本质是"好感解锁的选修课"，同样受 § 4 全部规则约束。

### 8.1 继承与已有资产

四层来源模型**原样继续有效**（`DATABASE_SOURCE_DOCTRINE.md`）：典籍见证 / 传统解释 / 现代边界 / 产品呈现；原文与转述分离；许可不明内容 `internal_only`（DB-001~009 全部保留）。

已验证资产盘点（截至 2026-07-11，见 `DATABASE_VALIDATION_LEDGER.md`）：

| 资产 | 状态 | 对游戏的意义 |
|------|------|--------------|
| 六爻确定性排盘 | ✅ 17,026 项检查 + 第二实现复核 0 mismatch（《卜筮正宗》锚定） | 第一章摇卦交互直接可用 |

### 8.2 跟随制审计（剧情牵引，替代全量审计）

流程：章节 KP 清单先行 → 只审本章 KP（原文核对 + accepted 判定）→ 剧本引用 `source_ref` → 抉择点 lint 校验。**审计跟着剧情走，剧情不等全库审计。** 全库横向扩审只在对应域开门时进行。

### 8.3 第一章典籍清单（六爻入门）

| 《增删卜易》（野鹤老人） | 解卦规则主口径 + 占例改编素材 | ctext 可定位 |
| 《卜筮正宗》（王洪绪） | 装卦规则（已是排盘引擎声明口径） | 已在 validation report 锚定 |
| 《火珠林》 | 三钱起卦法源流 | 补充见证 |

多来源冲突时记录差异与采纳决定，不覆盖（继承 DB-006）。第一章解卦口径以《增删卜易》为主、《卜筮正宗》为辅，冲突处从主口径并在 KP 中注记。

### 8.4 存储形态（防过度建设）

第一章 KP 总量 ≤ 5，知识库先落**结构化文件**（`database/knowledge/` 下 YAML/JSON，git 版本化即版本控制），不建 SQLite 表。迁移触发条件：KP > 50 或出现运行时查询需求。`tianji.db` 继续只管用户数据（users/history/config + 新增学习状态表）。

## 9. 第一章规格 —《第一卦》（垂直切片）

### 9.1 剧情梗概

玩家入门术数宗门，引路人（师姐/师兄，性别开局可选）带新人。门中失窃事件 → 引路人带玩家摇人生第一卦（真实六爻起卦交互）→ 解卦教学 → 玩家凭卦象在两条线索中抉择 → 真相揭晓 + 好感萌芽 + 宗门层面悬念。

### 9.2 知识点清单（≤3 新 KP）

| KP | 名称 | 主要出处 | 教学时刻覆盖 |
|----|------|----------|--------------|
| KP-LY-001 | 阴阳爻与卦的构成 | 《周易·系辞》 | 示范 → 引导 → 独立 |
| KP-LY-002 | 三钱摇卦法（六掷成卦） | 《火珠林》/《卜筮正宗》 | 引导（亲手摇） |
| KP-LY-003 | 动爻与变卦 | 《增删卜易》 | 示范 → 独立（抉择点主判定） |

隐性铺垫（不教不考）：五行名词在对话中自然出现，为筑基阶段埋线。

### 9.3 验收标准

1. 30–60 分钟通关；含 1 次完整摇卦交互（复用已验证六爻引擎）+ ≥2 个计分抉择点。
2. 每个计分抉择点通过 lint：KP 标注 + verdict + source_ref 完整，引用 KP 均 accepted。
3. **玩家测试**：通关者能拿三枚硬币独立起一卦并说出卦的基本结构（KP-001/002 达"掌握"的现实验证）。
4. 掌握度/灵力/好感三数值正确记录并可在存档中查看。
5. 引路人双性别变体均可完整通关，教学内容与判定一致。

## 10. AI 使用边界（继承 v0.1 § 11，口径不变）

AI 负责：NPC 对话风味变体、开放性对话（约束在已 accepted 知识上下文内）、复盘总结、氛围文案。
AI 不负责：排盘计算、知识事实生成、抉择点判定、渡劫通过裁定。
旧条款"AI 出题→AI 评判"继续作废；AI 生成的一切教学口径必须绑定 accepted KP。

## 13. 主要风险

| 风险 | 处理 |
|------|------|
| **剧本质量是生死线**（系统救不了平庸写作） | 剧本先行于代码；每章剧本独立评审后才进开发 |

| 双性别变体拖慢产能 | 骨架共享 + 模板变量；变体层只做风味不做分叉 |
| 解卦知识审计成为瓶颈 | 跟随制：每章只审 ≤3 个 KP，体量可控 |
| 又滑回"堆模块/堆基建" | 进度唯一口径 = 可玩章节数与玩家掌握结果，不是模块数（继承旧 PRD 风险表第一条） |
